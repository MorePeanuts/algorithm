## 题目描述
在一条单行道上，有 `n` 辆车开往同一目的地。目的地是几英里以外的 `target` 。

给定两个整数数组 `position` 和 `speed` ，长度都是 `n` ，其中 `position[i]` 是第 `i` 辆车的位置， `speed[i]` 是第 `i` 辆车的速度(单位是英里/小时)。

一辆车永远不会超过前面的另一辆车，但它可以追上去，并以较慢车的速度在另一辆车旁边行驶。

**车队** 是指并排行驶的一辆或几辆汽车。车队的速度是车队中 **最慢** 的车的速度。

即便一辆车在 `target` 才赶上了一个车队，它们仍然会被视作是同一个车队。

返回到达目的地的车队数量 。

**示例 1：**

**输入：** `target = 12, position = [10,8,0,5,3], speed = [2,4,1,1,3]`

**输出：** `3`

**解释：**

- 从 10（速度为 2）和 8（速度为 4）开始的车会组成一个车队，它们在 12 相遇。车队在 `target` 形成。
- 从 0（速度为 1）开始的车不会追上其它任何车，所以它自己是一个车队。
- 从 5（速度为 1） 和 3（速度为 3）开始的车组成一个车队，在 6 相遇。车队以速度 1 移动直到它到达 `target`。

**示例 2：**

**输入：** `target = 10, position = [3], speed = [3]`

**输出：** `1`

**解释：**

只有一辆车，因此只有一个车队。

**示例 3：**

**输入：** `target = 100, position = [0,2,4], speed = [4,2,1]`

**输出：** `1`

**解释：**

- 从 0（速度为 4） 和 2（速度为 2）开始的车组成一个车队，在 4 相遇。从 4 开始的车（速度为 1）移动到了 5。
- 然后，在 4（速度为 2）的车队和在 5（速度为 1）的车成为一个车队，在 6 相遇。车队以速度 1 移动直到它到达 `target`。

**提示：**

- `n == position.length == speed.length`
- `1 <= n <= 10^5`
- `0 < target <= 10^6`
- `0 <= position[i] < target`
- `position` 中每个值都 **不同**
- `0 < speed[i] <= 10^6`

## 题目解析
### 解法1
**排序 + 到达时间比较**：按位置降序排序，比较每辆车到达终点的时间来判断是否形成车队。

**原理：**
后面的车如果到达终点所需时间小于等于前面的车，说明后面的车会追上前面的车形成车队；否则后面的车会单独形成新车队。

**步骤：**
1. 建立位置到速度的映射，计算每辆车到达终点的时间
2. 按位置降序排序（从最靠近终点的车开始处理）
3. 遍历每辆车，比较其到达时间与当前车队领头车的到达时间
4. 若当前车到达时间更长，说明无法追上，形成新车队，更新领头车到达时间

**示例：**
- 输入：`target = 12, position = [10,8,0,5,3], speed = [2,4,1,1,3]`
- 按位置降序排列：`[10, 8, 5, 3, 0]`
- 计算到达时间：
  - 位置 10，速度 2：时间 = (12-10)/2 = 1.0（车队1）
  - 位置 8，速度 4：时间 = (12-8)/4 = 1.0 ≤ 1.0（加入车队1）
  - 位置 5，速度 1：时间 = (12-5)/1 = 7.0 > 1.0（车队2）
  - 位置 3，速度 3：时间 = (12-3)/3 = 3.0 ≤ 7.0（加入车队2）
  - 位置 0，速度 1：时间 = (12-0)/1 = 12.0 > 7.0（车队3）
- 输出：3 个车队

### 解法2
**桶排序优化**：利用桶排序避免使用 sort 库，在位置范围较小时可获得线性时间复杂度。

**原理：**
由于位置值的范围是 `[0, target)`，可以使用布尔数组作为桶来记录每个位置是否有车，然后从大到小遍历获得有序位置序列，避免显式排序。

**步骤：**
1. 创建长度为 target 的布尔数组作为桶，标记有车的位置
2. 建立位置到速度的映射
3. 从 target-1 到 0 倒序遍历桶，收集有车的位置（自动降序）
4. 按照解法1的逻辑比较到达时间，统计车队数量

**示例：**
- 输入：`target = 12, position = [10,8,0,5,3], speed = [2,4,1,1,3]`
- 桶数组：索引 0,3,5,8,10 处为 true
- 倒序遍历得到：`[10, 8, 5, 3, 0]`（与解法1排序结果相同）
- 后续计算过程与解法1相同，输出 3 个车队

> **注意**：当 target 值非常大时，该方法会占用大量内存，此时解法1更优；当 n 接近 target 时，该方法可达到 O(target) 时间复杂度。
