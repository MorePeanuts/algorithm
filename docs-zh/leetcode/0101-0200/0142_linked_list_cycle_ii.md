## 题目描述
给定一个链表的头节点  `head` ，返回链表开始入环的第一个节点。 *如果链表无环，则返回 `null`。*

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（**索引从 0 开始**）。如果 `pos` 是 `-1`，则在该链表中没有环。**注意：`pos` 不作为参数进行传递**，仅仅是为了标识链表的实际情况。

**不允许修改** 链表。

**示例 1：**

![](https://assets.leetcode.com/uploads/2018/12/07/circularlinkedlist.png)

```
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的链表节点
解释：链表中有一个环，其尾部连接到第二个节点。
```

**示例 2：**

![](https://assets.leetcode.cn/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist_test2.png)

```
输入：head = [1,2], pos = 0
输出：返回索引为 0 的链表节点
解释：链表中有一个环，其尾部连接到第一个节点。
```

**示例 3：**

![](https://assets.leetcode.cn/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist_test3.png)

```
输入：head = [1], pos = -1
输出：返回 null
解释：链表中没有环。
```

**提示：**

- 链表中节点的数目范围在范围 `[0, 104]` 内
- `-105 <= Node.val <= 105`
- `pos` 的值为 `-1` 或者链表中的一个有效索引

**进阶：**你是否可以使用 `O(1)` 空间解决此题？

## 题目解析
### 解法1
**方法名**：计算环长 + 双指针

**原理：**
首先使用快慢指针判断链表是否有环，若有环则计算环的长度。然后使用两个指针，一个指针从链表头部先走环长步数，另一个指针从头部出发，两者以相同速度前进，相遇点即为环的入口。

**步骤：**
1. 使用快慢指针（slow 每次走 1 步，fast 每次走 2 步）遍历链表，若两指针相遇则说明有环
2. 从相遇点开始，继续移动一个指针直到再次回到相遇点，统计环的长度
3. 让 fast 指针从头部先走环长步数
4. slow 指针从头部出发，与 fast 指针同时每次走 1 步
5. 当两指针相遇时，该节点即为环的入口

**示例：**
- 输入：`[3,2,0,-4]`，环从索引 1 开始
- 快慢指针相遇后，计算得环长为 3
- fast 从头部先走 3 步到达节点 -4
- slow 从头部开始，两者同时移动：
  - slow: 3 → 2 → 0
  - fast: -4 → 2 → 0
  - 相遇在节点 2，即环的入口

### 解法2
**方法名**：双指针两次相遇法

**原理：**
利用快慢指针两次相遇找到环的入口。第一次相遇后，将快指针重新指向链表头部，然后两个指针以相同速度前进，第二次相遇点即为环的入口。其核心数学原理是：从链表头部到环入口的距离等于从第一次相遇点到环入口的距离（可能加上若干倍环长）。

**第一次相遇的数学分析：**
设链表共有 \(a + b\) 个节点，其中：
- \(a\)：链表头部到环入口的节点数（不计入口节点）
- \(b\)：环的节点数

设两指针分别走了 \(f\)（fast）和 \(s\)（slow）步，则：
1. \(f = 2s\)（fast 速度是 slow 的 2 倍）
2. \(f = s + nb\)（fast 比 slow 多走了 \(n\) 个环的长度）

两式相减得：\(f = 2nb\)，\(s = nb\)

**第二次相遇的原理：**
如果让指针从链表头部一直向前走，到达环入口时的步数为 \(k = a + nb\)（先走 \(a\) 步到入口，之后每绕 \(b\) 步回到入口）。

目前 slow 已经走了 \(nb\) 步，因此让 slow 再走 \(a\) 步就可到达入口。

虽然我们不知道 \(a\) 的具体值，但可以利用：从链表头部到入口恰好需要走 \(a\) 步。因此，将 fast 重新指向头部，让两个指针以相同速度前进，当 fast 走了 \(a\) 步到达入口时，slow 也恰好走了 \(a + nb\) 步到达入口，两指针在入口处第二次相遇。

**步骤：**
1. 使用快慢指针（slow 每次走 1 步，fast 每次走 2 步）遍历链表
2. 若 fast 到达链表末端，说明无环，返回 null
3. 若 slow 与 fast 第一次相遇，说明有环，进入下一步
4. 将 fast 重新指向链表头部
5. slow 和 fast 同时每次向前走 1 步
6. 当两指针第二次相遇时，该节点即为环的入口

**示例：**
- 输入：`[3,2,0,-4]`，环从索引 1 开始（\(a = 1\)，\(b = 3\)）
- 第一次相遇：slow 和 fast 在节点 -4 相遇（slow 走了 3 步 = \(nb = 1 \times 3\)，fast 走了 6 步 = \(2nb = 2 \times 3\)）
- fast 重新指向头部节点 3
- 第二次相遇：
  - slow: -4 → 2
  - fast: 3 → 2
  - 相遇在节点 2，即环的入口
