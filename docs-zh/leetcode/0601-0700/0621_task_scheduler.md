## 题目描述
给你一个用字符数组 `tasks` 表示的 CPU 需要执行的任务列表，用字母 A 到 Z 表示，以及一个冷却时间 `n`。每个周期或时间间隔允许完成一项任务。任务可以按任何顺序完成，但有一个限制：两个 **相同种类** 的任务之间必须有长度为`n`的冷却时间。

返回完成所有任务所需要的 **最短时间间隔** 。

**示例 1：**

**输入：**tasks = ["A","A","A","B","B","B"], n = 2

**输出：**8

**解释：**

在完成任务 A 之后，你必须等待两个间隔。对任务 B 来说也是一样。在第 3 个间隔，A 和 B 都不能完成，所以你需要待命。在第 4 个间隔，由于已经经过了 2 个间隔，你可以再次执行 A 任务。

**示例 2：**

**输入：**tasks = ["A","C","A","B","D","B"], n = 1

**输出：**6

**解释：**一种可能的序列是：A -> B -> C -> D -> A -> B。

由于冷却间隔为 1，你可以在完成另一个任务后重复执行这个任务。

**示例 3：**

**输入：**tasks = ["A","A","A","B","B","B"], n = 3

**输出：**10

**解释：**一种可能的序列为：A -> B -> idle -> idle -> A -> B -> idle -> idle -> A -> B。

只有两种任务类型，A 和 B，需要被 3 个间隔分割。这导致重复执行这些任务的间隔当中有两次待命状态。

**提示：**

- `1 <= tasks.length <= 104`
- `tasks[i]` 是大写英文字母
- `0 <= n <= 100`

## 题目解析
### 解法1
**方法名**：最大堆 + 队列模拟调度

**原理：**
通过模拟 CPU 调度过程，使用最大堆优先调度剩余次数最多的任务，同时用队列管理处于冷却期的任务，确保相同任务之间的间隔满足冷却时间要求。

**步骤：**
1. 统计每个任务的出现频率
2. 将所有任务按频率构建最大堆
3. 使用一个长度为 `n+1` 的队列来管理冷却中的任务
4. 每个时间间隔：
   - 从队列头部取出一个任务（若有且剩余次数>0则放回堆）
   - 从堆中取出频率最高的任务执行
   - 执行后若该任务还有剩余次数，则加入队列尾部等待冷却
   - 若无任务可执行则进入待命状态
5. 累计时间直到所有任务完成

**示例：**
输入：tasks = ["A","A","A","B","B","B"], n = 2
- 初始：堆 [A(3), B(3)], 队列 [nil, nil, nil]
- 时间1：执行 A(2) → 队列 [nil, nil, A(2)]
- 时间2：执行 B(2) → 队列 [nil, A(2), B(2)]
- 时间3：待命 → 队列 [A(2), B(2), nil]
- 时间4：A 冷却完成，执行 A(1) → 队列 [B(2), nil, A(1)]
- 时间5：B 冷却完成，执行 B(1) → 队列 [nil, A(1), B(1)]
- 时间6：待命 → 队列 [A(1), B(1), nil]
- 时间7：A 冷却完成，执行 A(0) → 队列 [B(1), nil, nil]
- 时间8：B 冷却完成，执行 B(0) → 完成
- 总时间：8

### 解法2
**方法名**：贪心算法

**原理：**
采用贪心策略，优先安排出现次数最多的任务以确定调度的基本框架，然后将其他任务填充到框架的间隙中。当任务种类足够多时，可以完全填满间隙而无需待命时间。

**步骤：**
1. 统计每个任务的出现频率
2. 找出最大频率 `maxFreq` 和有多少个任务达到这个最大频率 `maxCount`
3. 计算基础框架所需时间：`(maxFreq - 1) * (n + 1) + maxCount`
4. 最终结果取上述计算值和任务总数的较大值（当任务种类很多时，可能不需要待命时间）

**示例：**
输入：tasks = ["A","A","A","B","B","B"], n = 2
- 最大频率 maxFreq = 3（A 和 B 各出现3次）
- maxCount = 2（有两个任务达到最大频率）
- 基础框架时间 = (3-1) * (2+1) + 2 = 2*3+2 = 8
- 任务总数 = 6
- 最终结果 = max(8, 6) = 8
