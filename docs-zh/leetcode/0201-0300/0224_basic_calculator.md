## 题目描述
给你一个字符串表达式 `s` ，请你实现一个基本计算器来计算并返回它的值。

注意:不允许使用任何将字符串作为数学表达式计算的内置函数，比如 `eval()` 。

**示例 1：**

```
输入：s = "1 + 1"
输出：2
```

**示例 2：**

```
输入：s = " 2-1 + 2 "
输出：3
```

**示例 3：**

```
输入：s = "(1+(4+5+2)-3)+(6+8)"
输出：23
```

**提示：**

- `1 <= s.length <= 3 * 105`
- `s` 由数字、`'+'`、`'-'`、`'('`、`')'`、和 `' '` 组成
- `s` 表示一个有效的表达式
- `'+'` 不能用作一元运算(例如， `"+1"` 和 `"+(2 + 3)"` 无效)
- `'-'` 可以用作一元运算(即 `"-1"` 和 `"-(2 + 3)"` 是有效的)
- 输入中不存在两个连续的操作符
- 每个数字和运行的计算将适合于一个有符号的 32位 整数

## 题目解析
### 解法1
**方法名**：中缀转后缀 + 后缀表达式求值

**原理：**
将中缀表达式转换为后缀表达式（逆波兰式），然后使用栈来计算后缀表达式的值。这种方法将运算符优先级问题转化为线性处理问题，简化了计算逻辑。

**步骤：**
1. **表达式分词**：将输入字符串转换为 token 列表（数字、运算符、括号），处理一元负号（在 `(` 前的 `-` 前补 0）
2. **中缀转后缀**：使用栈处理运算符优先级，遇到数字直接输出，遇到左括号入栈，遇到右括号则弹出栈顶直到左括号，遇到运算符则弹出栈中优先级大于等于当前的运算符后入栈
3. **后缀求值**：遍历后缀表达式，遇到数字入栈，遇到运算符弹出两个数计算后将结果入栈，最终栈顶即为答案

**示例：**
输入 `"(1+(4+5+2)-3)+(6+8)"`：
- 分词 → `["(", "1", "+", "(", "4", "+", "5", "+", "2", ")", "-", "3", ")", "+", "(", "6", "+", "8", ")"]`
- 转后缀 → `["1", "4", "5", "+", "2", "+", "+", "3", "-", "6", "8", "+", "+"]`
- 求值 → 23

**复杂度分析：**
- 时间复杂度：O(n)，三次线性遍历表达式
- 空间复杂度：O(n)，使用栈存储中间结果

> **注意**：一元负号处理是关键，需通过前一个非空白字符判断是否需要补 0

### 解法2
**方法名**：双栈直接求值（简化版）

**原理：**
在分词后的中缀表达式上直接使用双栈（数字栈 + 运算符栈）求值，避免显式转换为后缀表达式。根据运算符优先级和括号规则，在遍历过程中实时计算。

**步骤：**
1. **表达式分词**：同解法1，处理一元负号
2. **双栈遍历**：
   - 遇到数字：入数字栈
   - 遇到 `(`：入运算符栈
   - 遇到 `)`：弹出运算符并计算直到遇到 `(`
   - 遇到 `+/-`：弹出栈中所有可计算的运算符（直到 `(`）后入栈
3. **清算栈**：遍历结束后弹出剩余运算符并计算

**示例：**
输入 `"1 + 1"`：
- 分词 → `["1", "+", "1"]`
- 遍历 → 数字1入栈，`+` 入栈，数字1入栈
- 清算 → 弹出 `+` 和两个1，计算得2

**复杂度分析：**
- 时间复杂度：O(n)，每个 token 入栈和出栈各一次
- 空间复杂度：O(n)，双栈存储

> **注意**：由于只有 `+` 和 `-` 两种运算符且优先级相同，可简化运算符优先级判断逻辑

### 解法3
**方法名**：单遍扫描 + 双栈求值（优化版）

**原理：**
将分词和求值合并为单遍扫描，在遍历字符串的同时完成 token 识别和双栈求值，进一步优化空间和时间效率。

**步骤：**
1. **单遍扫描**：使用双指针识别数字 token，同时处理运算符和括号
2. **实时处理**：识别到完整数字后立即入栈，识别到运算符/括号时立即按规则处理栈
3. **清算栈**：同解法2

**示例：**
输入 `" 2-1 + 2 "`：
- 扫描到 `2`：入数字栈 [2]
- 扫描到 `-`：入运算符栈 [`-`]
- 扫描到 `1`：入数字栈 [2, 1]
- 扫描到 `+`：弹出 `-` 计算得 1 入栈，再入 `+`，栈状态：数字 [1]，运算符 [`+`]
- 扫描到 `2`：入数字栈 [1, 2]
- 清算：弹出 `+` 计算得 3

**复杂度分析：**
- 时间复杂度：O(n)，单遍扫描 + 栈操作
- 空间复杂度：O(n)，双栈存储

> **注意**：单遍扫描需小心处理空格和数字边界，确保数字 token 识别正确
