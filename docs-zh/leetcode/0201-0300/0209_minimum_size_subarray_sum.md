## 题目描述
给定一个含有 `n`个正整数的数组和一个正整数 `target` **。**

找出该数组中满足其总和大于等于`target`的长度最小的 **子数组** `[numsl, numsl+1, ..., numsr-1, numsr]` ，并返回其长度**。**如果不存在符合条件的子数组，返回 `0` 。

**示例 1：**

```
输入：target = 7, nums = [2,3,1,2,4,3]
输出：2
解释：子数组 [4,3] 是该条件下的长度最小的子数组。
```

**示例 2：**

```
输入：target = 4, nums = [1,4,4]
输出：1
```

**示例 3：**

```
输入：target = 11, nums = [1,1,1,1,1,1,1,1]
输出：0
```

**提示：**

- `1 <= target <= 109`
- `1 <= nums.length <= 105`
- `1 <= nums[i] <= 104`

**进阶：**

- 如果你已经实现`O(n)` 时间复杂度的解法, 请尝试设计一个 `O(n log(n))` 时间复杂度的解法。

## 题目解析
### 解法1
**方法名**：滑动窗口（双指针）

**原理：**
使用左右两个指针维护一个窗口，通过移动右指针扩展窗口使总和达到目标，再移动左指针缩小窗口以找到最小长度。

**步骤：**
1. 初始化左指针、右指针都指向数组起始位置，当前窗口和为第一个元素值，最小长度为数组长度+1（表示初始无符合条件的子数组）
2. 当右指针未越界且左指针不超过右指针时循环：
   - 若当前窗口和小于目标值，右移右指针并累加新元素到窗口和
   - 若当前窗口和大于等于目标值，更新最小长度，然后右移左指针并从窗口和中减去移出的元素
3. 最后根据最小长度是否仍为初始值判断是否存在符合条件的子数组

**示例：**
- 输入：target = 7, nums = [2,3,1,2,4,3]
- 处理过程：
  - 窗口 [2] → 和=2 < 7 → 扩展
  - 窗口 [2,3] → 和=5 < 7 → 扩展
  - 窗口 [2,3,1] → 和=6 < 7 → 扩展
  - 窗口 [2,3,1,2] → 和=8 ≥ 7 → 记录长度4，收缩 → [3,1,2]
  - 窗口 [3,1,2] → 和=6 < 7 → 扩展
  - 窗口 [3,1,2,4] → 和=10 ≥ 7 → 记录长度4，收缩 → [1,2,4]
  - 窗口 [1,2,4] → 和=7 ≥ 7 → 记录长度3，收缩 → [2,4]
  - 窗口 [2,4] → 和=6 < 7 → 扩展
  - 窗口 [2,4,3] → 和=9 ≥ 7 → 记录长度3，收缩 → [4,3]
  - 窗口 [4,3] → 和=7 ≥ 7 → 记录长度2，收缩 → [3]
- 输出：2

### 解法2
**方法名**：滑动窗口（优化版）

**原理：**
外层循环用右指针遍历数组，内层循环不断尝试收缩左指针，在单次遍历中完成窗口的扩展和收缩。

**步骤：**
1. 初始化左指针指向数组起始位置，当前窗口和为0，最小长度为数组长度+1
2. 右指针遍历数组，每次将当前元素加入窗口和
3. 当窗口和大于等于目标值时，循环执行：
   - 更新最小长度
   - 从窗口和中减去左指针指向的元素
   - 右移左指针
4. 最后根据最小长度是否仍为初始值判断是否存在符合条件的子数组

**示例：**
- 输入：target = 7, nums = [2,3,1,2,4,3]
- 处理过程：
  - 右指针到0，sum=2 → 不收缩
  - 右指针到1，sum=5 → 不收缩
  - 右指针到2，sum=6 → 不收缩
  - 右指针到3，sum=8 ≥ 7 → 记录长度4，收缩到sum=6（left=1）
  - 右指针到4，sum=10 ≥ 7 → 记录长度4，收缩到sum=7（left=2），记录长度3，收缩到sum=6（left=3）
  - 右指针到5，sum=9 ≥ 7 → 记录长度3，收缩到sum=7（left=4），记录长度2，收缩到sum=3（left=5）
- 输出：2
