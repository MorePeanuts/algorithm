## 题目描述
给你一个字符串表达式 `s` ，请你实现一个基本计算器来计算并返回它的值。

整数除法仅保留整数部分。

你可以假设给定的表达式总是有效的。所有中间结果将在 `[-231, 231 - 1]` 的范围内。

**注意：**不允许使用任何将字符串作为数学表达式计算的内置函数，比如 `eval()` 。

**示例 1：**

```
输入：s = "3+2*2"
输出：7
```

**示例 2：**

```
输入：s = " 3/2 "
输出：1
```

**示例 3：**

```
输入：s = " 3+5 / 2 "
输出：5
```

**提示：**

- `1 <= s.length <= 3 * 105`
- `s` 由整数和算符 `('+', '-', '*', '/')` 组成，中间由一些空格隔开
- `s` 表示一个 **有效表达式**
- 表达式中的所有整数都是非负整数，且在范围 `[0, 231 - 1]` 内
- 题目数据保证答案是一个 **32-bit 整数**

## 题目解析
### 解法1
**方法名**：中缀转后缀表达式（逆波兰式）求值

**原理：**
将中缀表达式（日常书写格式）转换为后缀表达式（运算符在操作数之后），消除运算符优先级的歧义，然后使用栈直接计算后缀表达式的值。

**步骤：**
1. **表达式解析**：将输入字符串解析为 tokens（数字和运算符），跳过空格
2. **中缀转后缀**：使用调度场算法（Shunting-yard algorithm）：
   - 遇到数字直接输出到后缀表达式
   - 遇到运算符，弹出栈中所有优先级大于等于当前运算符的运算符，然后将当前运算符入栈
   - `*` 和 `/` 优先级高于 `+` 和 `-`
3. **后缀表达式求值**：
   - 遇到数字入栈
   - 遇到运算符弹出两个数计算，结果入栈
   - 最终栈中剩余的数即为结果

**示例：**
- 输入: `"3+2*2"` 
- 解析 tokens: `["3", "+", "2", "*", "2"]`
- 转后缀: `["3", "2", "2", "*", "+"]`
- 求值过程: 3入栈 → 2入栈 → 2入栈 → 弹出2,2计算2*2=4入栈 → 弹出3,4计算3+4=7入栈
- 输出: `7`

### 解法2
**方法名**：两轮遍历（先乘除后加减）

**原理：**
利用运算符优先级，分两轮计算：第一轮先计算所有的乘法和除法，第二轮再计算加法和减法，每一轮都只处理同级运算的简单表达式。

**步骤：**
1. **表达式编码**：将字符串转换为整数数组，数字保持原值，运算符编码为负数（`+`=-1, `-`=-2, `*`=-3, `/`=-4）
2. **第一轮计算（乘除）**：遍历数组，遇到 `*` 或 `/` 时立即计算左右操作数并替换
3. **第二轮计算（加减）**：再次遍历数组，遇到 `+` 或 `-` 时计算左右操作数并替换
4. **结果**：数组最后剩余的一个元素即为表达式的值

**示例：**
- 输入: `"3+5/2"`
- 编码: `[3, -1, 5, -4, 2]`
- 第一轮（处理-4即`/`）: `[3, -1, 2]`
- 第二轮（处理-1即`+`）: `[5]`
- 输出: `5`

### 解法3
**方法名**：单栈实时计算

**原理：**
使用一个栈存储待累加的中间结果，遇到 `+` 和 `-` 时将带符号的数入栈，遇到 `*` 和 `/` 时立即与栈顶元素计算，最后将栈中所有数求和得到结果。

**步骤：**
1. **初始化**：空栈，记录上一个运算符（默认为 `+`）
2. **遍历表达式**：
   - 解析出完整数字
   - 根据上一个运算符决定如何处理当前数字：
     - `+`：数字直接入栈
     - `-`：数字取反入栈
     - `*`：弹出栈顶，与当前数相乘，结果入栈
     - `/`：弹出栈顶，除以当前数，结果入栈
   - 更新上一个运算符
3. **求和**：将栈中所有数相加即为最终结果

**示例：**
- 输入: `"3+2*2"`
- 处理数字3，上一个运算符`+`: 栈`[3]`
- 处理数字2，上一个运算符`+`: 栈`[3, 2]`
- 处理数字2，上一个运算符`*`: 弹出2，计算2*2=4，栈`[3, 4]`
- 求和: 3+4=7
- 输出: `7`
