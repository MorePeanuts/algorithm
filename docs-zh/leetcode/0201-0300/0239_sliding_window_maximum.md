## 题目描述
给你一个整数数组 `nums`，有一个大小为 `k`的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。

返回 *滑动窗口中的最大值* 。

**示例 1：**

```
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7
```

**示例 2：**

```
输入：nums = [1], k = 1
输出：[1]
```

**提示：**

- `1 <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`
- `1 <= k <= nums.length`

## 题目解析
### 解法1
**方法名**：单调队列

**原理：**
使用一个双端队列来维护滑动窗口内可能成为最大值的元素索引。队列中保持元素单调递减，这样队列头部始终是当前窗口的最大值。

**步骤：**
1. 初始化一个空双端队列
2. 处理前k个元素，构建初始的单调队列
3. 遍历数组剩余元素：
   - 移除队列头部超出窗口范围的元素
   - 从队列尾部移除所有小于当前元素的元素（因为它们不可能成为后续窗口的最大值）
   - 将当前元素加入队列尾部
   - 队列头部即为当前窗口的最大值，加入结果

**示例：**
输入 nums = [1,3,-1,-3,5,3,6,7], k = 3
- 窗口[1,3,-1]: 队列[3,-1] → 最大值3
- 窗口[3,-1,-3]: 队列[3,-1,-3] → 最大值3
- 窗口[-1,-3,5]: 队列[5] → 最大值5
- 窗口[-3,5,3]: 队列[5,3] → 最大值5
- 窗口[5,3,6]: 队列[6] → 最大值6
- 窗口[3,6,7]: 队列[7] → 最大值7
输出: [3,3,5,5,6,7]

**复杂度分析：**
- 时间复杂度：O(n)，每个元素最多入队和出队各一次
- 空间复杂度：O(k)，队列最多存储k个元素

> **注意**：
> - 队列中存储的是元素值，也可以存储索引来更精确地判断元素是否在窗口内
> - 当遇到相等元素时，不需要移除队列中的元素，因为它们在窗口中的位置不同
