## 题目描述
给定一个 `m x n` 二维字符网格 `board`和一个单词（字符串）列表 `words`， *返回所有二维网格上的单词* 。

单词必须按照字母顺序，通过 **相邻的单元格** 内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母在一个单词中不允许被重复使用。

**示例 1：**

![](https://assets.leetcode.com/uploads/2020/11/07/search1.jpg)

```
输入：board = [["o","a","a","n"],["e","t","a","e"],["i","h","k","r"],["i","f","l","v"]], words = ["oath","pea","eat","rain"]
输出：["eat","oath"]
```

**示例 2：**

![](https://assets.leetcode.com/uploads/2020/11/07/search2.jpg)

```
输入：board = [["a","b"],["c","d"]], words = ["abcb"]
输出：[]
```

**提示：**

- `m == board.length`
- `n == board[i].length`
- `1 <= m, n <= 12`
- `board[i][j]` 是一个小写英文字母
- `1 <= words.length <= 3 * 104`
- `1 <= words[i].length <= 10`
- `words[i]` 由小写英文字母组成
- `words` 中的所有字符串互不相同

## 题目解析

### 解法1：暴力DFS搜索（逐个单词检查）

**方法名**：对每个单词在矩阵中进行深度优先搜索

**原理：**
遍历每个单词，从矩阵的每个位置开始进行DFS搜索，检查当前路径是否能构成该单词。通过标记访问过的单元格避免重复使用同一单元格。

**步骤：**
1. 遍历单词列表中的每个单词
2. 对每个单词，从矩阵的每个位置 (i,j) 开始尝试
3. 使用DFS搜索：若当前字符匹配单词首字母，则标记为已访问并继续搜索四个方向
4. 若单词长度减至0，说明找到完整单词
5. 回溯：取消当前单元格的访问标记，供其他路径使用

**示例：**
以单词 "oath" 为例：
- 从 (0,0) 的 'o' 开始
- 向下到 (1,0) 的 'e'（不匹配 'a'，回溯）
- 向右到 (0,1) 的 'a'，匹配第二个字符
- 继续向下到 (1,1) 的 't'，匹配第三个字符
- 再向下到 (2,1) 的 'h'，匹配第四个字符
- 单词完整匹配，加入结果

> **注意**：此方法在单词列表较大时效率较低，因为每个单词都要遍历整个矩阵。

**复杂度分析：**
- **时间复杂度**：O(W × M × N × 4<sup>L</sup>)，其中 W 是单词数量，M×N 是矩阵大小，L 是单词的最大长度。每个单词需要遍历 M×N 个起点，每个起点最多进行 4<sup>L</sup> 次搜索。
- **空间复杂度**：O(M×N + L)。需要 M×N 的访问矩阵，递归栈深度最大为 L。

### 解法2：哈希表 + DFS（从矩阵生成单词）

**方法名**：从矩阵每个位置开始DFS生成可能的单词，用哈希表快速查找

**原理：**
将所有单词存入哈希表，然后从矩阵的每个位置开始进行DFS搜索，生成所有可能的路径字符串。每生成一个字符串就检查是否在哈希表中，若存在则加入结果并从哈希表中删除（避免重复）。

**步骤：**
1. 将所有单词存入哈希表 dictionary，同时记录单词的最大长度 maxLength
2. 从矩阵的每个位置 (i,j) 开始DFS搜索
3. 搜索过程中累积字符生成当前字符串
4. 若当前字符串在哈希表中，加入结果并从哈希表删除
5. 当路径长度达到 maxLength 时停止继续搜索
6. 回溯：移除最后一个字符并取消访问标记

**示例：**
从 (0,0) 的 'o' 开始：
- 生成 "o"（不在哈希表）
- 向右到 (0,1) 生成 "oa"（不在哈希表）
- 向右到 (0,2) 生成 "oaa"（不在哈希表）
- 向下到 (1,2) 生成 "oaae"（超过长度限制，回溯）
- ...
- 另一条路径：(0,0)→(0,1)→(1,1)→(2,1) 生成 "oath"（在哈希表中，加入结果）

> **注意**：此方法通过哈希表快速判断当前路径是否为目标单词，且通过删除已找到的单词避免重复，但在最坏情况下仍会生成大量无效路径。

**复杂度分析：**
- **时间复杂度**：O(M × N × 4<sup>L</sup>)，其中 M×N 是矩阵大小，L 是单词的最大长度。需要遍历 M×N 个起点，每个起点最多进行 4<sup>L</sup> 次搜索。哈希表的查找和删除是 O(1)。
- **空间复杂度**：O(W + M×N + L)。需要存储 W 个单词的哈希表，M×N 的访问矩阵，递归栈深度最大为 L。

### 解法3：字典树（Trie）+ DFS（前缀剪枝优化）

**方法名**：使用字典树存储单词前缀，在DFS过程中实时剪枝无效路径

**原理：**
将所有单词构建成字典树（Trie），字典树的每个节点表示一个前缀。在DFS搜索过程中，每走一步就检查当前前缀是否存在于字典树中，若不存在则立即剪枝，避免继续搜索无效路径。

**步骤：**
1. 将所有单词插入字典树 Trie，每个节点标记是否为单词结尾
2. 从矩阵的每个位置 (i,j) 开始DFS搜索
3. 搜索过程中累积字符生成当前字符串
4. 在字典树中查找当前字符串：
   - 若前缀不存在，立即剪枝返回
   - 若前缀存在且是完整单词，加入结果
5. 继续搜索四个方向（如果前缀仍存在）
6. 回溯：移除最后一个字符并取消访问标记

**示例：**
字典树包含 "oath", "pea", "eat", "rain"：
```
root
├── o -> o
│   └── a -> oa
│       └── t -> oat
│           └── h -> oath (end=true)
├── p -> p
│   └── e -> pe
│       └── a -> pea (end=true)
├── e -> e
│   └── a -> ea
│       └── t -> eat (end=true)
└── r -> r
    └── a -> ra
        └── i -> rai
            └── n -> rain (end=true)
```

从 (0,0) 开始搜索 "o" → "oa" → "oat" → "oath"（找到），而遇到前缀 "oaa" 时字典树中不存在，立即剪枝。

> **注意**：此方法通过字典树的前缀匹配实现高效剪枝，大幅减少无效搜索，是三种方法中最优的。找到单词后将 end 标记设为 false，可避免重复添加同一单词。

**复杂度分析：**
- **时间复杂度**：O(M × N × 4<sup>L</sup>)，其中 M×N 是矩阵大小，L 是单词的最大长度。虽然理论上界与解法2相同，但通过字典树剪枝，实际运行时间大幅减少，尤其在单词列表较大时优势明显。
- **空间复杂度**：O(W×L + M×N + L)。需要存储 W 个单词的字典树（每个单词最多 L 个节点），M×N 的访问矩阵，递归栈深度最大为 L。
