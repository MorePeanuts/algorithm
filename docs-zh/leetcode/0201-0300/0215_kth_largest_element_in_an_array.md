## 题目描述
给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。

请注意，你需要找的是数组排序后的第 `k` 个最大的元素，而不是第 `k` 个不同的元素。

你必须设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

**示例 1:**

```
输入: [3,2,1,5,6,4], k = 2
输出: 5
```

**示例 2:**

```
输入: [3,2,3,1,2,4,5,5,6], k = 4
输出: 4
```

**提示：**

- `1 <= k <= nums.length <= 105`
- `-104 <= nums[i] <= 104`

## 题目解析
### 解法1
**方法名**：最大堆

**原理：**
利用最大堆的性质，堆顶元素始终是最大值。通过构建最大堆并弹出 k 次，即可得到第 k 大的元素。

**步骤：**
1. 将整个数组构建成一个最大堆
2. 执行 k 次弹出操作
3. 第 k 次弹出的元素即为第 k 大的元素

**示例：**
- 输入: [3,2,1,5,6,4], k = 2 -> 构建最大堆 [6,5,4,2,3,1] -> 第一次弹出 6，第二次弹出 5 -> 输出: 5

### 解法2
**方法名**：最小堆（大小为 k）

**原理：**
维护一个大小为 k 的最小堆，堆顶元素是当前已遍历元素中的第 k 大元素。遍历剩余元素，若大于堆顶则替换。最终堆顶即为答案。

**步骤：**
1. 用数组前 k 个元素构建最小堆
2. 遍历数组剩余元素
3. 若当前元素大于堆顶，弹出堆顶并插入当前元素
4. 遍历完成后，堆顶元素即为第 k 大的元素

**示例：**
- 输入: [3,2,1,5,6,4], k = 2 -> 初始堆 [2,3] -> 处理 5（替换 2）-> 堆 [3,5] -> 处理 6（替换 3）-> 堆 [5,6] -> 处理 4（跳过）-> 堆顶 5 -> 输出: 5
