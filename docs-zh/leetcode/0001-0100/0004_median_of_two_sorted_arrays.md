## 题目描述
给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的 **中位数** 。

算法的时间复杂度应该为 `O(log (m+n))` 。

**示例 1：**

```
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3] ，中位数 2
```

**示例 2：**

```
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5
```

**提示：**

- `nums1.length == m`
- `nums2.length == n`
- `0 <= m <= 1000`
- `0 <= n <= 1000`
- `1 <= m + n <= 2000`
- `-106 <= nums1[i], nums2[i] <= 106`

## 题目解析
### 解法1
**方法名**：二分查找分割点

**原理：**
通过在较短数组中进行二分查找，找到一个分割点，使得两个数组被分割后，左边所有元素小于等于右边所有元素，且左右两边元素总数相等（或差1）。此时中位数可由分割点附近的元素计算得出。

**步骤：**
1. 确保 nums1 是较短的数组，若不是则交换，这样可以减少二分查找的次数
2. 初始化二分查找的左右边界 l=-1, r=len(nums1)
3. 在 nums1 中进行二分查找：
   - 计算 nums1 的分割点 p1 = (l + r) / 2
   - 根据左右元素总数相等的原则，计算 nums2 的分割点 p2 = (len1+len2+1)/2 - (p1+1) - 1
   - 若 nums1[p1] > nums2[p2+1]，说明 p1 太靠右，需要向左搜索，调整 r = p1
   - 否则，说明 p1 可行或可以向右搜索，调整 l = p1
4. 找到合适的分割点后，计算左边最大值 leftMax 和右边最小值 rightMin
5. 根据数组总长度的奇偶性返回中位数：
   - 奇数长度：返回 leftMax
   - 偶数长度：返回 (leftMax + rightMin) / 2

**示例：**
以 nums1 = [1,3], nums2 = [2] 为例：
- 总长度为 3（奇数），需要左边 2 个元素，右边 1 个元素
- 在 nums1 中二分查找，最终 p1=0（元素 1），p2=0（元素 2）
- leftMax = max(1, 2) = 2，rightMin = min(3, +∞) = 3
- 总长度为奇数，返回 leftMax = 2.0

以 nums1 = [1,2], nums2 = [3,4] 为例：
- 总长度为 4（偶数），需要左边 2 个元素，右边 2 个元素
- 在 nums1 中二分查找，最终 p1=1（元素 2），p2=0（元素 3）
- leftMax = max(2, 3) = 3？不，实际 p1=1 表示 nums1 左边有 2 个元素，p2=0 表示 nums2 左边有 1 个元素，总共 3 个？需要重新看算法。实际正确的分割是 p1=1, p2=-1：leftMax = max(2, -∞) = 2，rightMin = min(+∞, 3) = 3，返回 (2+3)/2 = 2.5

> **注意**：需要处理边界情况，如某个数组为空、分割点在数组两端等。代码中使用 math.MinInt 和 math.MaxInt 来处理这些边界情况。
