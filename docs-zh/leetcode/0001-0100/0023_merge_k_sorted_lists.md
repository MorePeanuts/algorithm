## 题目描述
给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

**示例 1：**

```
输入：lists = [[1,4,5],[1,3,4],[2,6]]
输出：[1,1,2,3,4,4,5,6]
解释：链表数组如下：
[
  1->4->5,
  1->3->4,
  2->6
]
将它们合并到一个有序链表中得到。
1->1->2->3->4->4->5->6
```

**示例 2：**

```
输入：lists = []
输出：[]
```

**示例 3：**

```
输入：lists = [[]]
输出：[]
```

**提示：**

- `k == lists.length`
- `0 <= k <= 10^4`
- `0 <= lists[i].length <= 500`
- `-10^4 <= lists[i][j] <= 10^4`
- `lists[i]` 按 **升序** 排列
- `lists[i].length` 的总和不超过 `10^4`

## 题目解析
### 解法1
**方法名**：分治法两两合并

**原理：**
利用分治思想，将 k 个链表的合并问题分解为多个两两合并的子问题，通过逐层合并最终得到一个有序链表。该方法借鉴了归并排序的思想，时间复杂度更优。

**步骤：**
1. 检查输入链表数组是否为空，为空则直接返回 nil
2. 当链表数组长度大于 1 时，循环进行合并：
   - 每次遍历数组，将相邻的两个链表合并为一个
   - 将合并后的链表放回数组前部
   - 截断数组，只保留合并后的部分
3. 最终数组中剩下的一个链表即为合并结果

**示例：**
- 输入：`[[1,4,5],[1,3,4],[2,6]]`
- 第一轮合并：合并 `[1,4,5]` 和 `[1,3,4]` 得到 `[1,1,3,4,4,5]`，`[2,6]` 直接保留
- 第二轮合并：合并 `[1,1,3,4,4,5]` 和 `[2,6]` 得到 `[1,1,2,3,4,4,5,6]`
- 输出：`[1,1,2,3,4,4,5,6]`

**复杂度分析：**
- 时间复杂度：O(N log k)，其中 N 是所有链表节点的总数，k 是链表的数量。每轮合并 O(N)，共需 O(log k) 轮
- 空间复杂度：O(1)，只使用常数级别的额外空间

> **注意**：在合并过程中，当链表数量为奇数时，最后一个链表无需配对，直接进入下一轮。
