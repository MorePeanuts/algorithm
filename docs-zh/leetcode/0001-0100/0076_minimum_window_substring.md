## 题目描述
给定两个字符串 `s` 和 `t`，长度分别是 `m` 和 `n`，返回 s 中的 **最短窗口 子串**，使得该子串包含 `t` 中的每一个字符（**包括重复字符**）。如果没有这样的子串，返回空字符串`""`。

测试用例保证答案唯一。

**示例 1：**

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
```

**示例 2：**

```
输入：s = "a", t = "a"
输出："a"
解释：整个字符串 s 是最小覆盖子串。
```

**示例 3:**

```
输入: s = "a", t = "aa"
输出: ""
解释: t 中两个字符 'a' 均应包含在 s 的子串中，
因此没有符合条件的子字符串，返回空字符串。
```

**提示：**

- `m == s.length`
- `n == t.length`
- `1 <= m, n <= 10^5`
- `s` 和 `t` 由英文字母组成

**进阶：**你能设计一个在 `O(m + n)` 时间内解决此问题的算法吗？

## 题目解析
### 解法1
**方法名**：滑动窗口 + 哈希表

**原理：**
使用滑动窗口在字符串 s 中寻找包含 t 所有字符的最短子串。通过两个哈希表分别记录 t 中字符的需求数量和当前窗口中缺少的字符，动态调整窗口的左右边界来找到最小窗口。

**步骤：**
1. 初始化两个哈希表：`count` 记录 t 中每个字符的需要数量，`miss` 记录当前窗口中缺少的字符
2. 遍历字符串 t，填充 `count` 和 `miss` 哈希表
3. 使用左右指针 l 和 r 表示滑动窗口的边界，初始都为 0
4. 移动右指针 r 扩展窗口，遇到 t 中的字符时更新 `count`，当某个字符的计数 ≤ 0 时从 `miss` 中移除
5. 当窗口包含所有所需字符时（`miss` 为空），尝试移动左指针 l 缩小窗口以找到更小的有效窗口
6. 在缩小窗口过程中，记录窗口长度最小的子串
7. 当窗口不再包含所有所需字符时，继续移动右指针

**示例：**
以 s = "ADOBECODEBANC", t = "ABC" 为例：
- 初始：count={A:1, B:1, C:1}, miss={A:true, B:true, C:true}
- r 移动到索引 0 (A)：count={A:0, B:1, C:1}, miss={B:true, C:true}
- r 移动到索引 3 (B)：count={A:0, B:0, C:1}, miss={C:true}
- r 移动到索引 5 (C)：count={A:0, B:0, C:0}, miss={}，此时窗口为 "ADOBEC"
- 开始缩小窗口，l 移动到 1，窗口变为 "DOBEC"，此时 miss={A:true}
- 继续移动 r，最终找到最小窗口 "BANC" (索引 9-12)

**复杂度分析：**
- 时间复杂度：O(m + n)，其中 m 是 s 的长度，n 是 t 的长度。每个字符最多被左右指针各访问一次。
- 空间复杂度：O(1)，因为字符集仅限于英文字母，最多 52 个不同字符，哈希表大小是常数。

> **注意**：
> 1. 需要处理 t 中包含重复字符的情况（如 t = "aa"）
> 2. 窗口的左右边界移动需要正确处理哈希表的更新
> 3. 初始最小长度设置为 len(s) + 1 可以确保找到有效窗口时一定会更新
