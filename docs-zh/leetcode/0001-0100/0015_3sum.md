## 题目描述
给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：** 答案中不可以包含重复的三元组。

**示例 1：**

```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。
```

**示例 2：**

```
输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。
```

**示例 3：**

```
输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0 。
```

**提示：**

- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

## 题目解析
### 解法1
**排序 + 双指针**：将三数之和问题转化为排序数组上的双指针查找问题。

**原理：**
先对数组排序，固定第一个数后，在剩余部分使用双指针从两端向中间逼近，利用有序性快速定位和为目标值的两个数。

**步骤：**
1. 对数组升序排序
2. 遍历数组，固定第一个数 `nums[i]`
3. 若 `nums[i] > 0`，后面不可能有三数之和为 0，提前终止
4. 跳过重复的 `nums[i]` 避免重复解
5. 双指针 `l = i+1`、`r = n-1`，寻找 `nums[l] + nums[r] == -nums[i]`
6. 根据和与目标的大小关系移动指针，找到解后同时移动并跳过重复值

**示例：**
- 输入 `[-1,0,1,2,-1,-4]` → 排序后 `[-4,-1,-1,0,1,2]`
- `i=0`，固定 `-4`，目标和为 `4`，双指针 `l=1, r=5`：
  - `nums[1]+nums[5] = -1+2 = 1 < 4`，`l++`
  - 继续移动，无法找到和为 4 的组合
- `i=1`，固定 `-1`，目标和为 `1`，双指针 `l=2, r=5`：
  - `nums[2]+nums[5] = -1+2 = 1 = 1`，找到 `[-1,-1,2]`，`l++, r--`
  - `nums[3]+nums[4] = 0+1 = 1 = 1`，找到 `[-1,0,1]`，`l++, r--`
  - `l >= r`，结束
- `i=2`，`nums[2] == nums[1]`，跳过避免重复
- `i=3`，固定 `0`，目标和为 `0`，双指针无法找到
- `i=4`，`nums[4] > 0`，提前终止
- 结果：`[[-1,-1,2], [-1,0,1]]`
