## 题目描述
给定一个整数数组 `temperatures` ，表示每天的温度，返回一个数组 `answer` ，其中 `answer[i]` 是指对于第 `i` 天，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 `0` 来代替。

**示例 1:**

```
输入: temperatures = [73,74,75,71,69,72,76,73]
输出: [1,1,4,2,1,1,0,0]
```

**示例 2:**

```
输入: temperatures = [30,40,50,60]
输出: [1,1,1,0]
```

**示例 3:**

```
输入: temperatures = [30,60,90]
输出: [1,1,0]
```

**提示：**

- `1 <= temperatures.length <= 10^5`
- `30 <= temperatures[i] <= 100`

## 题目解析

### 解法1

**单调递减栈**：利用单调栈快速找到每个元素右侧第一个更大的元素。

**原理：**
维护一个存储索引的单调递减栈，当遇到比栈顶温度更高的温度时，说明找到了栈顶元素的"下一个更高温度"，计算天数差即为答案。

**步骤：**
1. 初始化结果数组 `answer` 和空栈 `stack`（存储索引）
2. 遍历温度数组，对于每个温度 `t`：
   - 若栈非空且当前温度高于栈顶索引对应的温度，弹出栈顶，计算天数差填入答案
   - 重复上述步骤直到栈空或栈顶温度不小于当前温度
   - 将当前索引入栈
3. 遍历结束后，栈中剩余的索引对应位置答案保持为 0

**示例：**
- 输入 `[73,74,75,71,69,72,76,73]`
- 处理过程：
  - `i=0`: 栈空，入栈 `[0]`
  - `i=1`: 74>73，弹出0，`answer[0]=1`，入栈 `[1]`
  - `i=2`: 75>74，弹出1，`answer[1]=1`，入栈 `[2]`
  - `i=3`: 71<75，入栈 `[2,3]`
  - `i=4`: 69<71，入栈 `[2,3,4]`
  - `i=5`: 72>69，弹出4，`answer[4]=1`；72>71，弹出3，`answer[3]=2`；72<75，入栈 `[2,5]`
  - `i=6`: 76>72，弹出5，`answer[5]=1`；76>75，弹出2，`answer[2]=4`，入栈 `[6]`
  - `i=7`: 73<76，入栈 `[6,7]`
- 输出 `[1,1,4,2,1,1,0,0]`

### 解法2

**逆向遍历 + 跳跃优化**：从后往前遍历，利用已计算的答案进行跳跃查找。

**原理：**
从数组末尾向前遍历，对于每个位置，利用后续位置已经计算好的"等待天数"进行跳跃，快速定位到下一个更高温度的位置。

**步骤：**
1. 初始化结果数组 `answer`，从倒数第二个元素开始向前遍历
2. 对于每个位置 `i`，从 `j = i+1` 开始查找：
   - 若 `temperatures[j] > temperatures[i]`，找到答案 `answer[i] = j - i`
   - 若 `temperatures[j] <= temperatures[i]` 且 `answer[j] == 0`，说明 `j` 后面没有更高温度，`i` 也不会有
   - 否则跳跃到 `j += answer[j]`，利用已计算的答案加速查找
3. 遍历结束后返回 `answer`

**示例：**
- 输入 `[73,74,75,71,69,72,76,73]`
- 逆向处理：
  - `i=7`: 最后一个，`answer[7]=0`
  - `i=6`: j=7，73<76，`answer[6]=0`
  - `i=5`: j=6，76>72，`answer[5]=1`
  - `i=4`: j=5，72>69，`answer[4]=1`
  - `i=3`: j=4，69<71，跳跃 j=4+1=5；72>71，`answer[3]=2`
  - `i=2`: j=3，71<75，跳跃 j=3+2=5；72<75，跳跃 j=5+1=6；76>75，`answer[2]=4`
  - `i=1`: j=2，75>74，`answer[1]=1`
  - `i=0`: j=1，74>73，`answer[0]=1`
- 输出 `[1,1,4,2,1,1,0,0]`
