## Description
Given an integer array nums, return all the triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, and `j != k`, and `nums[i] + nums[j] + nums[k] == 0`.

Notice that the solution set must not contain duplicate triplets.

**Example 1:**

```
Input: nums = [-1,0,1,2,-1,-4]
Output: [[-1,-1,2],[-1,0,1]]
Explanation: 
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0.
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0.
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0.
The distinct triplets are [-1,0,1] and [-1,-1,2].
Notice that the order of the output and the order of the triplets does not matter.
```

**Example 2:**

```
Input: nums = [0,1,1]
Output: []
Explanation: The only possible triplet does not sum up to 0.
```

**Example 3:**

```
Input: nums = [0,0,0]
Output: [[0,0,0]]
Explanation: The only possible triplet sums up to 0.
```

**Constraints:**

- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

## Solution
### Approach 1
**Sorting + Two Pointers**: Transform the 3Sum problem into a two-pointer search on a sorted array.

**Principle:**
Sort the array first, fix the first number, then use two pointers from both ends moving toward the center to find pairs that sum to the target value.

**Steps:**
1. Sort the array in ascending order
2. Iterate through the array, fixing the first number `nums[i]`
3. If `nums[i] > 0`, no valid triplet exists, break early
4. Skip duplicate `nums[i]` to avoid duplicate solutions
5. Use two pointers `l = i+1`, `r = n-1` to find `nums[l] + nums[r] == -nums[i]`
6. Move pointers based on sum comparison, skip duplicates after finding a solution

**Example:**
- Input `[-1,0,1,2,-1,-4]` → After sorting `[-4,-1,-1,0,1,2]`
- `i=0`, fix `-4`, target sum `4`, pointers `l=1, r=5`:
  - `nums[1]+nums[5] = -1+2 = 1 < 4`, `l++`
  - Continue moving, no pair sums to 4
- `i=1`, fix `-1`, target sum `1`, pointers `l=2, r=5`:
  - `nums[2]+nums[5] = -1+2 = 1 = 1`, found `[-1,-1,2]`, `l++, r--`
  - `nums[3]+nums[4] = 0+1 = 1 = 1`, found `[-1,0,1]`, `l++, r--`
  - `l >= r`, done
- `i=2`, `nums[2] == nums[1]`, skip to avoid duplicates
- `i=3`, fix `0`, target sum `0`, no valid pair found
- `i=4`, `nums[4] > 0`, early termination
- Result: `[[-1,-1,2], [-1,0,1]]`
